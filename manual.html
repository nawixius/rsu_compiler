Rust-u Programming Language Manual v1.0

Introduction to Rust-u

Rust-u is a modern pseudocode language that compiles to native Rust, combining high-level readability with Rust's performance and safety guarantees.

Key Features
Design Philosophy
Community
  • Intuitive syntax: Designed to read like pseudocode
  • Flexible type system: With inference and optional typing
  • Automatic memory management: Built-in borrow checker
  • Rust interoperability: Use any Rust crate
  • Cross-platform: Compiles to the same targets as Rust

Rust-u follows three core principles:

  1. Accessibility: Beginner-friendly syntax
  2. Power: Capable of complex system development
  3. Transparency: Generated code is readable Rust

Quick Start Guide

Installation

# Install from crates.io
cargo install rustu-compiler

# Verify installation
rustu --version

First Program

Rust-u
// hello.rsu
pub func main() {
    var name = "World"
    print("Hello, {name}!")
}
Generated Rust
// hello.rs
fn main() {
    let name = "World";
    println!("Hello, {}!", name);
}

Compilation and Execution

# Compile and run
rustu run hello.rsu

# Compile only
rustu compile hello.rsu

# Show generated Rust code
rustu show hello.rsu

Detailed Syntax Reference

Variables and Constants

var x = 10              // Type inference
var mut counter = 0     // Mutable variable
const PI = 3.1416       // Constant
global MAX_USERS = 100  // Global variable

The compiler automatically deduces when to use & or &mut based on program flow.

Advanced Functions

// Function with optional parameters
func greet(name: str, title: str = "Mr./Ms.") {
    print("Hello, {title} {name}")
}

// Generic function
func first<T>(list: list[T]) -> T {
    ret list[0]
}

// Closure
var square = func(x: int) -> int { x * x }

Structures and Enums

// Simple structure
obj Point(x: float, y: float) {
    func distance(&self) -> float {
        ret sqrt(self.x.pow(2) + self.y.pow(2))
    }
}

// Enumeration
enum Status {
    Active,
    Inactive,
    Suspended(reason: str)
}

// Usage
var p = Point(3.0, 4.0)
var status = Status::Suspended("Payment pending")

Common Patterns

Error Handling
OOP Patterns
Concurrency
// Explicit Result
func divide(a: float, b: float) -> Result<float, str> {
    if b == 0 {
        ret Err("Division by zero")
    }
    ret Ok(a / b)
}

// Usage with match
match divide(10.0, 0.0) {
    Ok(result) => print("Result: {result}"),
    Err(e) => print("Error: {e}")
}
// Builder pattern
obj ClientBuilder(name: str) {
    var mut age: int? = null
    var mut email: str? = null
    
    func with_age(age: int) -> Self {
        self.age = age
        ret self
    }
    
    func build() -> Client {
        ret Client {
            name: self.name,
            age: self.age ?: 18,
            email: self.email
        }
    }
}

// Usage
var client = ClientBuilder("Anna")
    .with_age(30)
    .build()
// Simple threads
var thread = spawn {
    print("Secondary thread executing")
}

// Channels
var (sender, receiver) = channel()

spawn {
    sender.send("Message from thread")
}

var message = receiver.receive()

Optimization Tips

  • Use const for immutable values
  • Prefer str over string when ownership isn't needed
  • Use list::with_capacity when you know the size in advance
